# Governance

Clippo starts as a single-maintainer project.

## Decision Making

- Small implementation decisions can be made in pull requests.
- Architecture, privacy, storage, packaging, and platform support decisions should be captured in ADRs or docs.
- v1 scope changes should update the roadmap and validation matrix.

## Release Authority

Until maintainership expands, the maintainer owns release approval, versioning, and signing.

## Issue Triage

Issues should be labeled by platform, type, severity, and whether they affect Maccy parity.

## Community Expectations

Contributors should keep discussions technical, respectful, and focused on the project goals.
