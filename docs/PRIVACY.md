# Privacy Model

Clippo is local-first and should not send clipboard contents over the network by default.

## Defaults

- No telemetry by default.
- No clipboard contents in logs by default.
- Diagnostics should redact clipboard contents by default.
- Users should be able to pause capture or ignore only the next copy.
- Users should be able to clear history quickly.

## Ignored Content

Clippo starts with Maccy-compatible confidential pasteboard type defaults and adds platform-sensitive format names for Windows and Linux where available.

macOS defaults include:

- `org.nspasteboard.TransientType`
- `org.nspasteboard.ConcealedType`
- `org.nspasteboard.AutoGeneratedType`
- `Pasteboard generator type`
- `com.agilebits.onepassword`
- `com.typeit4me.clipping`
- `de.petermaurer.TransientPasteboardType`
- `net.antelle.keeweb`

## Release Review

Every public release should include a privacy review covering clipboard capture, storage, logs, diagnostics, permissions, and packaging.

## Threat Model

Clippo's main privacy risks are:

- Capturing sensitive clipboard contents unintentionally.
- Persisting sensitive clipboard contents longer than the user expects.
- Logging clipboard contents through diagnostics, crashes, or debug output.
- Replaying a clipboard item into the wrong focused application.
- Reading clipboard data from password managers or transient system sources.
- Exposing stored history through insecure file permissions or backups.

v1 mitigations:

- Maccy-compatible ignored pasteboard types.
- Windows and Linux sensitive format hooks where platform metadata exists.
- Pause capture and ignore-next-copy controls.
- Local-only storage by default.
- Emergency clear-history-and-quit command.
- Redacted diagnostics policy.
- Permission and platform limitation docs before public release.

## Diagnostics

The shared core exposes diagnostic snapshots that report metadata such as history item count and capture state. They intentionally omit clipboard item preview text and full clipboard contents.
