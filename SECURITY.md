# Security Policy

Clippo is not production-ready yet. Security reports are still welcome during development.

## Reporting a Vulnerability

Until a dedicated private disclosure channel exists, please avoid posting clipboard-content vulnerabilities publicly with sensitive examples. Open a GitHub security advisory once the repository is hosted, or contact the maintainer through the repository contact method.

## Security Priorities

- Clipboard contents must not be logged by default.
- Diagnostics must redact clipboard contents by default.
- Release artifacts should be signed or checksummed before v1.
- Permission requirements must be documented clearly per OS.
